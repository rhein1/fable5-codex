# Code of Conduct

Be direct, technical, and respectful.

This project exists to improve evidence-first software engineering workflows. Keep discussions focused on reproducible behavior, source-backed evidence, and concrete improvements.

Do not post secrets, private keys, access tokens, personal data, or customer-sensitive evidence in issues, PRs, examples, or screenshots.

Maintainers may edit, hide, or remove content that exposes sensitive material, derails technical review, or creates legal/security risk for users.
