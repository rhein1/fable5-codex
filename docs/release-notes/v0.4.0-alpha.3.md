# Fable-5 for Codex v0.4.0-alpha.3

This release hardens installation, wrapper compatibility, package validation, CI, and benchmark provenance.

## Highlights

- Strict installer option parsing rejects typos, duplicates, split values, and invalid help combinations before any destination is selected or changed.
- PowerShell and Bash wrappers require Codex CLI `0.144.0` or newer for GPT-5.6 and support an explicit Codex executable.
- Package validation runs through Node on Windows, macOS, and Linux; CI covers Node 18 and Node 24.
- CI also installs the generated npm tarball, reruns its tests and validator, checks installer dry-run behavior, and reports one aggregate `Release gate`.
- Benchmark arms use separate newly created Codex homes and an external read-only fixture workspace.
- Benchmark workers strip caller secret variables, disable model-shell environment inheritance, reject linked runtime ancestors beneath the verified OS temp-root boundary, use private GUID-named runtimes, and remove copied auth before publication.
- The plugin arm installs the exact repository plugin and records its version and SHA-256 digest; the baseline arm contains no Fable marketplace or plugin.
- Failed, timed-out, empty, partial, changed, or stale trials cannot update public summaries or charts.
- Per-run and publication locks prevent concurrent runners from sharing runtime state or interleaving the latest artifact set.
- Selective resume requires exact run attestation, including prior summary and output digests, and supports temporary results on a different Windows drive than the checkout.
- Dependency-free Node chart rendering works across the CI operating-system matrix; complete chart, manifest, and latest publication is staged and rollback-protected with the authoritative latest-run pointer written last.
- Windows PowerShell BOM-prefixed scorer input is accepted, and workspace paths are redacted from public output even when slash styles are mixed.
- Render-only regeneration derives the model label from the attested summary and rejects mismatched run configuration.
- Actions are commit-SHA pinned, and Dependabot covers npm and GitHub Actions dependencies.
- The npm package includes `SECURITY.md` with the private vulnerability reporting path.

## Benchmark Status

The existing `20260713T234332Z` charts are retained as historical pre-alpha.3 workflow evidence. Alpha.3 does not claim a new benchmark result; a complete run with the hardened harness is required before replacing those charts.
